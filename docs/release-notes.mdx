

## Release Notes

**Tax API Updates**

* **Customer Tax Properties Example Update:** Adjusts the customer tax properties example to use a more descriptive format, enhancing clarity and helping developers avoid parsing issues. [View PR](https://github.com/bigcommerce/docs/pull/928)

* **Platform-Wide Availability Guidance:** Updates the documentation to clarify the separate process and business arrangement required for platform-wide tax provider availability, reducing confusion and unnecessary communication. [View PR](https://github.com/bigcommerce/docs/pull/924)

* **New `tax_properties` Field:** Introduces a `tax_properties` field in the customer object for Tax Provider API requests, enabling more detailed tax configurations. Note that this requires configuration via the Tax Properties API. [View PR](https://github.com/bigcommerce/docs/pull/923)

* **`type` Field in Tax Properties:** Adds an optional `type` field to tax properties, allowing specification of whether a property is associated with a product or a customer, defaulting to `PRODUCT`. [View PR](https://github.com/bigcommerce/docs/pull/922)

**B2B Documentation Enhancements**

* **Reusable Response Models:** Introduces new models to streamline B2B content, reducing repetitive code and enhancing documentation efficiency. [View PR](https://github.com/bigcommerce/docs/pull/921)

* **Quotes Documentation Refresh:** Updates the Server-to-Server Quotes documentation with new descriptive content and previously missing fields, improving clarity for developers. [View PR](https://github.com/bigcommerce/docs/pull/913)

* **Non-Catalyst Channel Setup Update:** Clarifies that the article is intended for non-Catalyst frameworks and includes a link to Catalyst-specific instructions, providing clearer guidance for integration. [View PR](https://github.com/bigcommerce/docs/pull/861)

**API Documentation Fixes**

* **Category Tree Request Documentation:** Adds a missing URL parameter in the `GET /api/v2/categories/tree` request documentation, ensuring complete and accurate information. [View PR](https://github.com/bigcommerce/docs/pull/920)

* **Invoice Update Endpoint:** Adds information and examples for the `customField` object in the Update an Invoice endpoint, enhancing developer understanding of its usage. [View PR](https://github.com/bigcommerce/docs/pull/919)

* **Customer Metafields Response Correction:** Fixes the API documentation for `GET /v3/customers/{customerId}/metafields` and `GET /v3/customers/{customerId}/metafields/{metafieldId}` responses, ensuring accurate response details. [View PR](https://github.com/bigcommerce/docs/pull/917)

**General Documentation Improvements**

* **Clarification of `extraFields` Parameter:** Provides clearer examples and usage information for the `extraFields` parameter in the Get All Companies endpoint, aiding developers in proper implementation. [View PR](https://github.com/bigcommerce/docs/pull/914)
